# 设计基础知识库

本文件包含设计基准方法论、预设风格模板选择、配色技术约束等核心设计知识，是所有设计场景的共享知识基础。

---

## 设计哲学

你不仅仅是一个编码员；你是一名**设计工程师 (Design Engineer)**。你的目标是构建让人感觉是"精心打磨 (Crafted)"而不是"简单拼凑 (Assembled)"的软件。

**核心使命**：创造**令人难忘的、有个性的**设计系统，而非千篇一律的「AI 默认风格」。

**反 AI Slop 原则**：

- 避免使用千篇一律的字体和陈词滥调的配色（紫色渐变、默认蓝）
- 避免过度设计：工具类不需要复杂导航，游戏类不需要 Footer，报告类不需要 Sidebar
- 避免千篇一律的布局和缺乏语境的设计

当你收到通用请求（如"让它美观"、"做一个现代仪表盘"）时：

1. **拒绝平庸**：不要默认使用标准 Bootstrap/Material 布局
2. **确立艺术指导**：选择一种特定风格并贯彻到底
3. **系统化思维**：在实现组件之前，先定义好字体层级、间距节奏和颜色交互规则

## 设计基准法（Design Benchmarking）

**核心理念**：不是套用固定规则，而是从内容理解推导设计语言。

### 工作流程

```
内容分析 → 内容理解 → 设计语言推导 → 规范输出
```

1. **深度理解内容**：受众是谁？要传达什么信息？期望的阅读情绪是什么？
2. **通过内容理解框架**形成对内容的全面理解
3. **推导设计语言**：基于内容理解的结果，推导配色、字体、间距、组件风格
4. **保持一致性**：同一份文档内所有设计决策都服务于内容理解的结论

### 内容理解框架（4 维度）

**受众维度**：

- 目标用户是谁？（专业人士/普通消费者/特定群体）
- 他们的预期和心理状态？（严肃决策/休闲浏览/学习探索）
- 阅读环境？（办公室/移动端/公开演示）

**内容维度**：

- 信息的正式程度？（官方发布/内部协作/社交分享）
- 信息密度？（数据密集/叙述为主/图文并重）
- 核心目的？（说服/告知/引导行动/建立信任）

**情感维度**：

- 期望唤起什么情绪？（信任/兴奋/安心/专注）
- 应该避免什么感受？（焦虑/无聊/困惑/廉价感）

**品牌维度**：

- 有既定品牌调性需要延续吗？
- 需要与什么竞品区分？

### 设计推导思维演示

> 以下示例展示的是**思考过程**，而非可直接套用的答案。

**示例：季度财务报告**

1. **回答内容理解问题**：
   - 受众：管理层，需要快速获取关键信息做决策
   - 正式程度：高，代表公司官方数据
   - 信息密度：高，数字密集
   - 期望情绪：信任、专注、清晰
   - 避免感受：混乱、不专业、数据不可信

2. **从理解推导设计方向**：
   - 「快速获取关键信息」→ 视觉层级要分明，重点数据需突出
   - 「代表公司官方」→ 设计语言需稳重、有权威感
   - 「数字密集」→ 需要等宽数字保证对齐，表格/图表清晰
   - 「信任感」→ 色彩选择需沉稳，避免过于鲜艳或随意

3. **转化为具体设计**：
   - 根据以上推导，自主选择符合方向的配色、字体、布局
   - 注意：同样是「稳重权威」，可以是深蓝、深绿、深灰、纯黑+红色点睛等多种选择 — 不要默认选择「冷灰+暖色强调」

## 设计师审美标准

| 审美维度 | 可操作的实践                                                                                                    | 避免               |
| -------- | --------------------------------------------------------------------------------------------------------------- | ------------------ |
| **比例** | 标题与正文字号比 >= 1.5x（如 `font-size: 1.875rem` : `font-size: 1rem`）；间距遵循 8pt 倍数                     | 随意的间距和尺寸   |
| **对比** | 主标题 `font-weight: 700`/`font-weight: 800` vs 正文 `font-weight: 400`/`font-weight: 300`；色彩对比度 >= 4.5:1 | 平均分布、缺乏主次 |
| **韵律** | 统一的卡片圆角/阴影/间距；章节标题样式保持一致                                                                  | 每个部分都不一样   |
| **平衡** | 网格布局保持视觉重心居中；大面积色块配合留白                                                                    | 头重脚轻或视觉失衡 |
| **留白** | section 间距 >= 2rem (32px)；卡片内边距 >= 1.5rem (24px)                                                        | 填满每一个像素     |
| **克制** | 装饰元素 <= 2 种手法；强调色使用面积 < 10%                                                                      | 装饰过度、元素堆砌 |

## 设计决策优先级

设计生成前必须明确三要素：

1. **Purpose**（解决什么问题）
2. **Tone**（美学方向）
3. **Differentiation**（独特记忆点）

**文档更新原则**：已存在规范时，提取现有设计基因，保持品牌调性一致性。

## 配色技术约束

> 适用于所有路径（模板 / 自定义），作为质量底线。

- 有色背景必须指定文字颜色，对比度 >= 4.5:1（大文本 >= 3:1）
- 避免纯黑 #000 / 纯白 #fff，使用 Off-white 或 Tinted Black（极简风格除外）

### 语义颜色（可选，按需定义）

| 用途        | 色相方向                        |
| ----------- | ------------------------------- |
| 成功/上升   | 绿色系 (H: 120-160)             |
| 警告/中风险 | 橙黄色系 (H: 30-50)，注意对比度 |
| 错误/下降   | 红色系 (H: 0-10)                |

### 图表配色（可选，有数据可视化需求时定义）

- **单系列**：使用模板主色或强调色
- **多系列**：从模板主色出发，通过色相偏移（+/-30-60 度）生成 3-6 个系列色
- **对比数据**：使用互补色（色相差 180 度）或语义色（涨/跌）

## 预设风格模板库

### 模板选择流程（必须按序执行）

**第 1 步：确定视觉服务目标**

> **关键原则**：三维标签描述的是你期望页面呈现的**视觉氛围**，而非内容本身的社会属性。一份面向高管的销售仪表盘可能需要「大胆张扬 + 冷峻」的高对比暗底来让 KPI 数据跳出屏幕；一份竞品分析报告可能需要「温暖 + 适度」的氛围深度来支撑沉浸式比较阅读。**不要将"面向管理层"等同于"必须选克制内敛的模板"。**

基于内容分析结果，从以下三个维度确定**期望的视觉氛围**：

| 维度     | 选项                                                                   | 判定依据                                                                                 |
| -------- | ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| 视觉能量 | 轻松活泼 / 中性平衡 / 沉稳庄重                                         | 页面需要多少视觉张力？数据密集型仪表盘往往需要高能量让指标醒目，深度报告可能需要沉稳氛围 |
| 色温方向 | 冷峻（科技感、精密感、数据感） / 中性 / 温暖（人文感、品质感、亲和力） | 内容本身的气质倾向，以及期望唤起的情绪                                                   |
| 表达力度 | 克制内敛 / 适度 / 大胆张扬                                             | 视觉装饰和风格化程度。克制=留白多、装饰少；大胆=强风格、高辨识度                         |

**常见内容类型 → 视觉方向速查**（非强制，仅辅助判断）：

| 内容类型 | 典型视觉需求 | 推荐视觉方向 |
|---------|------------|-------------|
| 数据仪表盘 / 看板 | 多指标需醒目、快速扫描 | 高对比深色底让数据发光，或极简浅色底让层级分明 |
| 项目管理 / 周报 | 多维状态并行、时间线清晰 | 精密仪表感或结构化网格 |
| 分析报告 / 复盘 | 沉浸阅读、叙事纵深 | 氛围感深色或编辑式排版 |
| 技术文档 / API | 开发者友好、代码可读 | 科技感深色或清晰极简 |
| 品牌 / 营销 | 视觉冲击、记忆点 | 高饱和撞色或个性化风格 |

---

**第 2 步：匹配模板**

将第 1 步得到的三维标签与下方模板列表中的「适配特征」比对，选择**特征重合度最高**的模板。当多个模板重合度相同时，阅读「视觉个性」和「适合的内容特征」列，选择**视觉个性与数据展示需求最贴合**的那个。

> **重要**：「适合的内容特征」列中的示例是典型场景，不是排他限制。一个模板能服务的场景远多于示例所列。

**第 3 步：验证合理性**

问自己两个问题：

1. 「这个视觉风格能让内容中最重要的信息更突出吗？」
2. 「如果目标受众看到这个风格，会觉得贴切还是违和？」

如果任一答案为否，退回第 2 步重新选择。

### 选择策略

**必须选择一个模板**。按三维标签匹配最佳模板后，READ 模板文件提取视觉风格（配色、字体、组件样式），结合 scene 文件的布局规范进行开发。用户指定色彩时，在选定模板的框架内替换配色即可。

**反偏向规则**：

- **禁止将任何模板视为"安全默认选项"** — 26 个模板地位平等，不存在兜底模板。如果你发现自己选某个模板的理由是"不确定选什么，这个最稳妥"，说明你的第 1 步调性判定不够明确，应退回重新分析内容
- 当多个模板同时匹配时，优先选择**视觉个性与内容主题关联更紧密**的那个，而非"最不会出错"的那个

**选择禁忌**：

- 禁止因为「上次用了 X 效果不错」而重复选择
- 禁止因为方案名称暗示的「适用场景」而预判（名称仅为标识符）
- 浅色/深色应根据内容需要选择，深色方案不是「备选」
- **禁止用"专业""极客"等泛化词作为选择极端风格的理由** — 大多数内容都是"专业"的，这个词不构成区分度
- **风格极端的模板（如 Cyberpunk HUD、Editorial Brutalism）需要内容主题的强支撑** — 例如 Cyberpunk HUD 适合安全/运维/黑客主题，不适合一般的"技术分享"或"数据分析"

### 风格执行规则（强制）

> **核心原则：style-preset 是设计规范，不是灵感参考。选定后必须忠实执行，不得自由发挥。**

**必须严格遵循 preset 文件的定义**：

| 参数类别 | 执行要求 |
|---------|---------|
| 配色方案（色值、色彩角色） | 以 preset 的 7-8 色角色系统为基准。可根据内容需要扩展同色系色阶、补充图表系列色和语义色，但新增色必须与 preset 主色体系协调 |
| 字体（字族、字重、字号层级） | 使用 preset 指定的字体和层级体系 |
| 圆角 / 阴影 / 边框 | 使用 preset 定义的具体 CSS 值 |
| 组件样式（卡片、按钮、标签等） | 遵循 preset 中的组件风格描述 |
| 动效 / 交互反馈 | 如 preset 有定义则照做，未定义的按「动效基础原则」补充 |

**仅以下情况允许偏离 preset**：

1. **用户明确要求修改**（如"把主色换成蓝色"）— 仅改用户指定的部分，其余保持不变
2. **preset 未覆盖的参数**（如 preset 未定义图表配色）— 从 preset 主色出发推导，风格保持一致
3. **无障碍对比度不达标** — 微调明度使对比度 >= 4.5:1，保持色相不变

**禁止的自由发挥行为**：

- 禁止替换 preset 的主色 / 强调色 / 背景色等核心角色色（扩展色阶和补充功能色除外）
- 禁止"觉得圆角太大/太小"而自行调整
- 禁止"觉得字体不适合内容"而替换 preset 指定的字体
- 禁止混合两个 preset 的风格元素
- 禁止在 preset 已定义的参数上"微调优化"

## 字体原则

> 选中模板后，使用模板自带的字体规范。以下仅为通用约束。

- 优先使用 Web 安全字体或 Google Fonts，确保多终端兼容
- 中西文混排时确保视觉重心一致
- 标题与正文字体搭配需层级清晰

## 动效基础原则

- 专注高影响力时刻，动效服务于体验
- 每个可交互元素需有 hover/active 反馈

## 全局布局契约

> 适用于所有场景（Dashboard、Prototype、info_viz 等），各场景不再重复定义。

- **全局容器 (Global Shell)**：响应式内边距，具体值由 code agent 根据场景判断
- **页面内容 (Page Content)**：填充父容器，**禁止**自行设置外边距
- **最大宽度**：根据场景选择合理的 `max-width` 值（如 `max-width: 72rem`、`max-width: 80rem` 等）
- **禁止**：`width: 100%` 让内容在大屏幕上无限延伸；大屏幕上内容贴边显示

> 完整 HTML 编码强制规范详见项目 `README.md`。

## 主题选择规则（强制）

深色/浅色由所选 style-preset 的「主题」列决定，具体配色值以 preset 文件内的定义为准。

- **禁止**混用深浅背景（如深色 Header + 浅色内容区）
- 用户未指定偏好时，优先选择浅色主题的模板
