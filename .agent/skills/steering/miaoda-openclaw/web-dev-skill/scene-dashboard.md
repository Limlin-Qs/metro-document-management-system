# Dashboard 场景开发规范（dashboard）

> **本文档是 Dashboard 场景的权威开发流程规范。** 严格按照下方定义的两个阶段逐步执行，不可跳过或合并任何步骤。

---

## 场景约束

- **布局**: 单页（单路由，非单文件），Dashboard Header → KPI 卡片 → 图表网格。各区块按独立 Section 组件拆分
- **导航**: 仅 Dashboard Header（分析标题 + 副标题可选），无 Sidebar/Topbar
- **功能按钮**: 除翻页控制外，禁止添加任何功能性操作按钮（导出、分享、筛选器等）

#### DB-2. Header 设计（强制）

> Dashboard 必须包含 Header，不可省略。

**Dashboard Header 结构**：
Header 采用通栏布局 (`width: 100%`)，内部包含一个居中容器 (max-width 居中)。容器内分为左右两个区域：

- **左侧区域**: 分析标题 (大字号、粗体) 和可选的副标题/描述 (小字号、浅色)
- **右侧区域** (可选): 元数据容器，如日期、状态标签等，使用胶囊样式展示

> 设计理念：仪表盘是针对数据的静态分析展示，Header 以「分析标题」为核心，不承载交互操作。

**导航规则**：
- 使用 Dashboard Header（分析标题 + 副标题可选）
- 默认不需要 Sidebar（除非明确是多页面 Dashboard 系统）
- 默认不需要 Topbar 导航

**禁止交互操作**：
- 禁止「筛选」（静态快照无过滤意义）
- 禁止「刷新」（数据为静态快照）
- 禁止「导出/下载」（数据来自用户已有文件）
- 禁止「分享」（非独立产品）

**Header 样式参考**：
- **通栏**：`width: 100%`，无圆角，内部内容与下方内容容器同宽居中
- **背景**：渐变/纹理优先（推荐），或使用页面背景色（Mode F 无背景）
- **标题字重**：`font-weight: 800` ~ `font-weight: 900` + `letter-spacing: -0.025em`
- **副标题字重**：`font-weight: 300`，与标题形成明显反差
- **元数据**（可选）：容器化处理（半透明背板 / 胶囊标签），右对齐或标题下方
- **内边距**：`padding: 0.75rem~1.25rem 1rem~1.5rem`
- **布局**：标题区左对齐，可选右侧元数据区

#### DB-3. 布局与图表系统

**布局结构**：
```
Header（通栏，`width: 100%`）
    |
max-width 居中容器 {
    以下区块按数据叙事逻辑自由组合，顺序不固定：
    - 指标卡片（关键 KPI 概览）
    - 图表（趋势/对比/占比等可视化）
    - 数据表格（明细数据、排行榜、对比表）
    - 文字分析（总结段落、洞察说明、趋势解读）
    - AI 洞察块（数据驱动的自动洞察）
}
```

> 页面布局应**内容驱动**：根据数据叙事需要自由组合区块类型和排列方式。重要内容给予更多空间（如全宽），次要内容可并列。

> Dashboard 推荐 `max-width: 80rem (1280px)` 起步，根据数据密度适当加宽（如 `max-width: 1400px`、`max-width: 1600px`），避免超过 `1600px`。

**图表尺寸自适应原则**：

| 判断维度 | 给全宽（`grid-column: 1 / -1`） | 可并列（半宽或更小） |
|---------|-------------------------|-------------------|
| **时间跨度** | 时间点 >= 12 | 时间点 < 7 |
| **数据系列** | 多系列（>= 3 条线/柱） | 单系列或双系列 |
| **类目数量** | 类目 >= 8 | 类目 < 5 |
| **分析重要性** | 核心分析结论的主图表 | 辅助补充型图表 |

**高度指导**：
- 默认图表高度 `height: 256px` ~ `height: 320px`
- 类目数多的横向条形图/表格：按类目数量增加高度
- 趋势图 + 多系列：建议 `height: 320px` ~ `height: 384px`

**容器宽度与图表数量联动**：
- 图表总数 <= 4：`max-width: 80rem (1280px)`（默认）
- 图表总数 5-8：考虑 `max-width: 1400px`
- 图表总数 > 8：考虑 `max-width: 1600px`，或分组分段展示

**图表配色原则**：
- **整体和谐**：图表色与主题配色视觉和谐即可，不必强制从主题色衍生
- **系列色**：选择与主题搭配和谐的色相，系列间保持饱和度/明度一致
- **语义色**：上升用绿色、下降用红色（固定语义，不随配色方案变化）

**图表选型指南**：

| 分析目的 | 推荐图表 |
|---------|---------|
| **趋势变化** | 折线图、面积图 |
| **类目对比** | 柱状图 |
| **占比分布** | 环形图 |
| **进度/完成率** | 进度条、进度环 |
| **明细数据** | 表格 |

> 其他图表（漏斗、雷达、散点、热力图等）根据需求选用。

#### DB-4. 组件与视觉效果

**Dashboard 核心组件**：

**指标卡片 (KPI Card)**：
- 数值突出显示 + 标签使用 muted 色
- 趋势箭头: 上升绿色 / 下降红色
- 布局: 响应式排列，列数和方式根据指标数量灵活决定

**图表容器 (Chart Card)**：
- 背景：使用配色方案中的 surface 色
- 圆角：`border-radius: 0.5rem` 或 `border-radius: 0.75rem`
- 内边距：`padding: 1rem` ~ `padding: 1.5rem`
- 阴影：`box-shadow: 0 1px 2px 0 rgba(0,0,0,0.05)`
- 标题：`font-size: 1rem; font-weight: 500`，位于左上角

**AI 洞察块**：
- 容器: 卡片样式，左侧 3px 彩色边框突出
- 图标: 灯泡/趋势图标
- 格式: 洞察标题(加粗) + 数据支撑(正常) + 建议(muted)

**数据表格**：
- 斑马纹背景（可选）
- Hover 高亮行
- 表头：`font-weight: 500`，背景略深
- 单元格：`font-size: 0.875rem`

> 应用 DB-1 选定的 style-preset 中定义的视觉参数（圆角/阴影/间距）。
